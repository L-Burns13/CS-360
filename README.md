# CS-360
Mobile Architect &amp; Programming

App Overview and User Needs
The goal of this app was to create a simple mobile inventory tracking application. The app allows users to log in, view inventory items, add new items, edit quantities, delete items, and receive notifications when inventory levels are low. It was designed for users who need an easy way to track inventory without complex features.

Screens, Features, and UI Design
The app includes a login screen, an inventory list screen, and screens for adding and editing inventory items. The UI was designed with simplicity in mind by using clear labels, readable text, and straightforward navigation. These design choices helped keep the app easy to use and focused on the user’s needs.

Coding Approach and Strategies
When developing the app, I focused on keeping the code organized and breaking functionality into smaller parts. Using separate activities, adapters, and model classes helped make the code easier to understand and maintain. This approach can be reused in future projects to support scalability and easier debugging.

Testing and Functionality
Testing was done using the Android emulator to ensure that features such as logging in, adding items, updating inventory, and sending notifications worked correctly. Testing was important because it helped identify issues early and confirmed that the app functioned as expected.

Challenges and Problem Solving
One challenge was making sure all parts of the app worked together correctly, especially the connection between the user interface and the app logic. I addressed this by testing features incrementally and reviewing how data moved through the app.

Areas of Success
I was most successful in implementing the core inventory features and connecting them to the user interface. This demonstrated my understanding of mobile app structure, user centered design, and basic Android development practices.
